# 2026-04-24 3090 vLLM Baseline

This note records the first clean `vllm` baseline for the Alienware host after
the strong-route cutover.

## Environment

- host: Alienware RTX 3090 (`24 GB`)
- container runtime: rootless Podman
- `vllm` image: `docker.io/vllm/vllm-openai:v0.19.1`
- serving pattern: one-model mode (`fast` and `strong` do not stay active at the
  same time)

## Validated profiles

### Fast

- model: `Qwen/Qwen2.5-7B-Instruct`
- context: `32768`
- route: `homelab-fast`
- result: local `/v1/models` OK, public model gateway chat completion OK

### Strong

- model: `Qwen/Qwen2.5-14B-Instruct-AWQ`
- context: `32768`
- route: `homelab-strong`
- extra tuning:
  - FP8 KV cache
  - prefix caching
  - chunked prefill
  - bounded scheduler settings
- result: local `/v1/models` OK, public model gateway chat completion OK

## Important findings

1. The older `latest` image on this host was not the runtime we wanted. Pinning
   `v0.19.1` was required to get onto a predictable CLI and startup path.
2. The old `--swap-space` flag is no longer the right tuning knob on this
   runtime. The newer stack exposes explicit CPU and KV offload controls.
3. `Qwen/Qwen2.5-14B-Instruct-AWQ` is a clean `32K` strong baseline on this GPU.

## Failed or incomplete paths

### Older runtime behavior

On the pre-upgrade runtime, the strong service repeatedly failed before reaching
an API state. The visible failure modes included:

- engine initialization instability
- misleading GPU-memory-utilization failures during startup

### 32B AWQ attempt

`Qwen/Qwen2.5-32B-Instruct-AWQ` did not become a reliable default on this
hardware. Even after KV-cache tuning, it did not stabilize cleanly enough to
keep as the default strong route.

## Promotion bar for the next experiment

A future candidate should only replace the current strong baseline if it:

- fits and starts reliably on the same hardware
- answers through both the local endpoint and the public gateway
- improves real homelab tasks, not just generic benchmarks
- keeps latency and operator complexity acceptable
